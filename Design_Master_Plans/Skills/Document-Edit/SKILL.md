# Document Edit Skill
## Use when: Editing any SD file in the Design Bible

### Before editing
1. Read SHATTERED_DOMINION_CONTEXT.md in full
2. Identify which canonical facts (Section 3) the edit touches
3. Check if the edit involves any Open Decision (Section 4) — if yes, STOP and flag it to Kyle before proceeding

### While editing
- Never change canonical facts without explicit instruction from Kyle
- Never use "Sylvarai" — always "Vaelari"
- Never reference four acts or four endings
- Never describe Ironkin or Drakelings as playable

### After editing
1. Increment the document version number (v1.0 → v1.1)
2. Add a change log entry at the very top of the file:
   > **v[X.X] — [Date] — [What changed and why]**
3. Update SHATTERED_DOMINION_CONTEXT.md Section 12 
   (Current Project Status) to reflect the change if it 
   resolves an open decision or completes a listed task

### Never do
- Resolve an open decision silently
- Delete content — move superseded content to an 
  Archive section within the file
- Edit SD_01 without explicit instruction — it is the source of truth
